# 🤝 贡献指南 | Contributing Guide

感谢你对 **美观实用新标签页（Elegant New Tab）** 的关注和贡献！  
Thank you for your interest in contributing to **Elegant New Tab**.

我们欢迎任何形式的贡献，包括但不限于代码、文档、设计建议和问题反馈。

---

## 📌 如何贡献 | How to Contribute

### 1️⃣ Fork 本仓库 | Fork the Repository
- 点击右上角 **Fork**
- 将项目 Fork 到你的 GitHub 账号下

---

### 2️⃣ 创建新分支 | Create a Branch

请基于 `main` 分支创建新的功能分支：

```bash
git checkout -b feature/your-feature-name
3️⃣ 编码规范 | Coding Guidelines
JavaScript

使用 原生 JavaScript（不引入额外框架）

保持代码简洁、可读

使用 const / let，避免 var

函数命名应语义清晰

CSS

避免使用全局污染样式

使用系统字体（禁止引入第三方字体）

保持样式层级清晰

HTML

语义化标签优先

不引入外部 JS / CSS CDN

4️⃣ 功能要求 | Feature Requirements

请确保你的修改：

✅ 不影响现有功能

✅ 不破坏新标签页加载速度

✅ 不添加广告、追踪或统计代码

✅ 不引入需要密钥的第三方服务

5️⃣ 提交前检查 | Before Submitting

在提交 Pull Request 前，请确认：

 插件可以正常加载（chrome://extensions）

 新标签页无 Console 报错

 天气 / 壁纸 / 时间功能不受影响

 修改内容有清晰的提交说明

6️⃣ 提交 Pull Request | Submit a Pull Request

推送你的分支到 GitHub

创建 Pull Request 到 main 分支

请清楚描述你做了什么修改，以及修改原因

🐞 问题反馈 | Reporting Issues

如果你发现 Bug 或有改进建议：

打开 Issues

说明：

浏览器类型与版本

操作系统

复现步骤

预期行为 vs 实际行为

请尽量提供清晰的信息，方便快速定位问题。

🎨 设计与体验 | Design & UX

本项目注重：

简洁

实用

性能

隐私

请避免：

过度动画

复杂配置

干扰用户体验的设计
